# LinkNest V1 开发推进

本文档把方案书中的开发顺序映射到当前仓库状态。

## 当前已完成

1. 初始化 Go 模块和目录结构。
2. 服务端配置加载、SQLite 连接、迁移执行、`/healthz`。
3. 用户注册、登录、当前用户、JWT 鉴权。
4. 设备注册、设备列表、WebSocket 心跳、离线扫描。
5. 文件上传闭环：`init-upload`、`missing-chunks`、单分片上传、`complete`、文件下载。
6. CLI 的认证、设备初始化、设备注册、设备列表、心跳、上传、下载、任务恢复命令。
7. 全目录 README 和基础运行文档。

## 下一步建议

1. 把静态首页扩展为登录页、设备页、文件页和任务页。
2. 增加服务层、HTTP API 和 CLI 恢复场景测试。
3. 为下载补充 Range 支持，为上传补充更细粒度进度反馈。
4. 优化本地任务缓存清理策略和失败重试体验。
